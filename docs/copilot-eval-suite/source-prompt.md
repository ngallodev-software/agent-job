# Create Copilot-Run Evaluation Suite for Measuring `agent-job` Value

You are working in the repository:

`invoke-codex-from-claude`

The project has pivoted from a Codex-specific runner to a universal `agent-job` workflow.

The production target is a Copilot-only or Copilot-primary environment.

The core supported Copilot workflow is:

```bash
agent-job validate <job.job.yaml>
agent-job render <job.job.yaml> --target copilot
agent-job package <job.job.yaml> --target copilot
agent-job report <package-dir>
```

Important constraint:
agent-job does not execute Copilot directly.
The purpose of this task is to create a set of Copilot-run evals.
The evals should be designed so a human can run them inside Copilot Chat, Copilot Workspace, or an approved Copilot-style environment.
The evaluation question is:

Does using an agent-job Copilot package improve Copilot-assisted engineering work compared with giving Copilot a normal competent markdown prompt?

Do not create a dashboard.
Do not add telemetry.
Do not add direct Copilot execution.
Do not add external network dependencies.
Do not fake Copilot results.
Do not make the baseline intentionally bad.
Do not design evals that make agent-job win unfairly.

Core Evaluation Design
The eval suite should be run by Copilot.
For each eval task, provide two prompts:

Baseline Copilot Prompt

A competent plain markdown prompt.

Represents what a human would normally paste into Copilot.

Agent-Job Copilot Prompt

Generated by agent-job package --target copilot.

Uses the structured job package, constraints, allowed paths, acceptance criteria, and report template.

Copilot should perform the same task under both conditions.
Then Copilot should complete an evaluator form comparing:

output quality

scope control

safety

evidence quality

test reporting

reviewability

ambiguity reduction

operator friction

whether agent-job added meaningful value

The suite should support manual A/B testing:
Run baseline prompt in Copilot.Capture Copilot output and diff.Reset repo/fixture.Run agent-job package prompt in Copilot.Capture Copilot output and diff.Ask Copilot to fill out evaluation form.Human reviews final comparison.

2. Task Set
Create at least 6 eval tasks.
The tasks should be realistic and small.
Each task should test a different aspect of value added by agent-job.
Task 1: Docs Consistency
Purpose:
Measure whether agent-job improves scoping and reviewability for a low-risk docs update.
Task idea:
Review active docs and update stale references so the current Copilot package workflow is described consistently.
Value dimensions:

command consistency

documentation accuracy

reduced stale references

evidence/reporting quality

Task 2: Small Bugfix
Purpose:
Measure whether agent-job helps Copilot perform a bounded bugfix with clearer acceptance criteria.
Task idea:
Fix a small known bug in a fixture function or parser helper.
Value dimensions:

task clarity

testability

changed-file discipline

test reporting

Task 3: Small Refactor
Purpose:
Measure whether agent-job prevents scope creep during a small refactor.
Task idea:
Refactor one small helper function for readability without changing behavior.
Value dimensions:

allowed path compliance

no unrelated rewrites

preserving behavior

human review clarity

Task 4: Test Addition
Purpose:
Measure whether agent-job produces better testing discipline.
Task idea:
Add one focused test for an existing validation or rendering behavior.
Value dimensions:

test specificity

whether Copilot explains tests run/not run

acceptance criteria tracking

Task 5: Scope Boundary Stress Test
Purpose:
Measure whether agent-job reduces unauthorized changes.
Task idea:
Make a change in an allowed file while explicitly avoiding a tempting adjacent forbidden file.
Value dimensions:

forbidden path respect

no broad cleanup

path policy awareness

safety language usefulness

Task 6: Ambiguous Request Stress Test
Purpose:
Measure whether agent-job reduces ambiguity compared with a normal prompt.
Task idea:
Improve the report output.
Baseline should be competent but less structured.
Agent-job should turn this into a bounded, acceptance-criteria-driven task.
Value dimensions:

ambiguity reduction

clarification of success criteria

reduced overengineering

reportability

1. Evaluator Prompt
Each task must include:
evaluator-prompt.md
This prompt is pasted into Copilot after both baseline and agent-job runs.
It asks Copilot to evaluate the two outputs.
The evaluator prompt must instruct Copilot to be skeptical and to score both workflows.
It should say:

Each evaluator prompt should be slightly customized for the task.
